# GUÍA PARA LA PRESENTACIÓN DEL TERCER PARCIAL

## Estructura de la Presentación (15-20 minutos)

### 1. INTRODUCCIÓN (2 minutos)
**Qué decir:**
- "Buenos días/tardes, mi nombre es [Tu Nombre]"
- "Hoy presentaré mi proyecto final: Sitio Web del Club América"
- "Este proyecto integra todo lo aprendido en el curso"

**Diapositiva:**
- Título del proyecto
- Tu nombre
- Fecha
- Logo del Club América

---

### 2. PROBLEMÁTICA Y OBJETIVO (2 minutos)
**Qué decir:**
- "La problemática que identifiqué fue la falta de un sitio centralizado con información del Club América"
- "El objetivo es crear una plataforma informativa accesible para los aficionados"
- "Permite conocer la historia, jugadores y establecer contacto"

**Diapositiva:**
- Problemática identificada
- Objetivo del proyecto
- Beneficios para los usuarios

---

### 3. TIPO DE SITIO Y JUSTIFICACIÓN (2 minutos)
**Qué decir:**
- "Elegí desarrollar un sitio web estático informativo"
- "Es la mejor opción porque el contenido es principalmente informativo"
- "No requiere actualizaciones en tiempo real"
- "Fácil de mantener y bajo costo de hosting"

**Diapositiva:**
- Tipo de sitio: Estático Informativo
- Ventajas de esta elección
- Comparación con otras opciones

---

### 4. TECNOLOGÍAS FRONTEND (3 minutos)
**Qué decir:**
- "Para el frontend utilicé HTML5 y CSS3"
- "HTML5 para la estructura del contenido"
- "CSS3 para los estilos y diseño visual"
- "Implementé diseño responsive para diferentes dispositivos"

**Diapositiva:**
- Logos de HTML5 y CSS3
- Características implementadas
- Principios UI/UX aplicados

**DEMO EN VIVO:**
- Mostrar la página principal
- Navegar entre secciones
- Mostrar responsive design (cambiar tamaño de ventana)

---

### 5. ESTRUCTURA DEL PROYECTO (2 minutos)
**Qué decir:**
- "El proyecto consta de 4 páginas principales"
- "Inicio, Jugadores, Historia y Contacto"
- "Todas comparten un diseño consistente"
- "Navegación intuitiva entre páginas"

**Diapositiva:**
- Mapa de navegación
- Estructura de archivos
- Capturas de cada página

---

### 6. BACKEND PROPUESTO (2 minutos)
**Qué decir:**
- "Para el backend propongo usar Node.js con Express"
- "Permite usar JavaScript tanto en frontend como backend"
- "Facilita la creación de APIs REST"
- "Gran comunidad y documentación"

**Diapositiva:**
- Arquitectura propuesta
- Tecnologías: Node.js, Express
- Diagrama de flujo

---

### 7. BASE DE DATOS (2 minutos)
**Qué decir:**
- "Seleccioné MongoDB como base de datos"
- "Es NoSQL, flexible y escalable"
- "Fácil integración con Node.js"
- "Gratuito en MongoDB Atlas"

**Diapositiva:**
- Logo de MongoDB
- Diseño de colecciones
- Ejemplo de documento JSON

---

### 8. REPOSITORIO GITHUB (2 minutos)
**Qué decir:**
- "El proyecto está alojado en GitHub"
- "Incluye README completo con documentación"
- "Control de versiones con Git"
- "Código abierto para revisión"

**Diapositiva:**
- Captura del repositorio en GitHub
- URL del repositorio
- Estadísticas (commits, archivos, etc.)

**DEMO EN VIVO:**
- Mostrar el repositorio en GitHub
- Mostrar el README
- Mostrar los commits

---

### 9. HOSTING Y DESPLIEGUE (2 minutos)
**Qué decir:**
- "Utilicé GitHub Pages para el hosting"
- "Es gratuito y fácil de configurar"
- "Incluye SSL automáticamente"
- "El sitio está disponible en [URL]"

**Diapositiva:**
- Logo de GitHub Pages
- URL del sitio publicado
- Ventajas del hosting seleccionado

**DEMO EN VIVO:**
- Mostrar el sitio en producción
- Verificar que funciona correctamente

---

### 10. PRUEBAS Y VALIDACIONES (2 minutos)
**Qué decir:**
- "Realicé pruebas exhaustivas del sitio"
- "Validé HTML y CSS con W3C"
- "Probé en diferentes navegadores"
- "Verifiqué responsive design"

**Diapositiva:**
- Checklist de pruebas realizadas
- Resultados de validación W3C
- Capturas en diferentes dispositivos

---

### 11. CONCLUSIONES Y PRÓXIMOS PASOS (2 minutos)
**Qué decir:**
- "Logré crear un sitio funcional y atractivo"
- "Aprendí sobre desarrollo web frontend"
- "Próximos pasos: implementar backend"
- "Agregar más funcionalidades"

**Diapositiva:**
- Logros del proyecto
- Aprendizajes clave
- Mejoras futuras
- Agradecimientos

## PREGUNTAS FRECUENTES Y RESPUESTAS

### P: ¿Por qué elegiste HTML y CSS en lugar de un framework?
**R:** "Elegí HTML y CSS puros para entender los fundamentos del desarrollo web. Los frameworks son útiles, pero es importante dominar las bases primero. Además, para un sitio estático informativo, HTML y CSS son suficientes y más ligeros."

### P: ¿Cómo manejarías la actualización de contenido?
**R:** "Actualmente es manual editando los archivos HTML. En el futuro, implementaría un CMS (Content Management System) o un panel de administración con backend para facilitar las actualizaciones sin tocar código."

### P: ¿Por qué MongoDB y no MySQL?
**R:** "MongoDB es NoSQL y más flexible para datos no estructurados. Se integra fácilmente con Node.js y JavaScript. Para este proyecto, donde la estructura de datos puede cambiar, MongoDB es más adecuado."

### P: ¿Cómo garantizas la seguridad del sitio?
**R:** "Actualmente uso HTTPS con el certificado SSL de GitHub Pages. Para el futuro backend, implementaría JWT para autenticación, bcrypt para contraseñas, validación de inputs y rate limiting."

### P: ¿El sitio es responsive?
**R:** "Sí, implementé diseño responsive usando CSS con media queries y meta viewport. El sitio se adapta a diferentes tamaños de pantalla: desktop, tablet y móvil."

### P: ¿Cuánto tiempo te tomó el proyecto?
**R:** "El desarrollo del frontend tomó aproximadamente [X] horas, incluyendo diseño, codificación y pruebas. La documentación y preparación para GitHub tomó [Y] horas adicionales."

### P: ¿Qué fue lo más difícil del proyecto?
**R:** "Lo más desafiante fue [menciona algo específico, ej: 'hacer que el diseño fuera responsive' o 'organizar el código de manera limpia']. Lo resolví investigando y probando diferentes enfoques."

### P: ¿Qué agregarías si tuvieras más tiempo?
**R:** "Agregaría un sistema de noticias dinámico, galería de imágenes, integración con redes sociales, y un sistema de comentarios. También implementaría el backend completo con Node.js y MongoDB."
